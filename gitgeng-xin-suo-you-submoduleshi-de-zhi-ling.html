git更新所有submodule時的指令

Sun 15 September 2013
By LucasWei

git在submodule的管理中，為了讓開發團隊的程式相依性能夠一致，所以當你clone一個專案下來，並且使用git submodule update --init的時候，其實你只會把他submodule更新到和 專案當初在管理時，使用的submodule版本而已，而不一定是最新的版本。

當然如果以開發軟體的角度來看這樣的特性是好的，可以確保所有共同開發的人都使用相同的版本，但是我今天在管理自己的vim的時候，因為底下有許多plug-in仍然在更新， 所以我想要從github上更新自己有在使用的plug-in，但是如果每個plug-in要更新，難道只能使用

cd bundle/plug-in1
git clone origin master
cd ../plug-in2
git ...

就顯得很麻煩，記得當初在修課的時候有提到xargs，前一次更新的時候就使用這個功能，利用shell來撰寫，不過也險得很多此一舉， 今天上網查了以後，參考了stackoverflow上的這篇，發現其實git裡面就有這個貼心的設計，所以特別做一下筆記，紀錄一下，其實只要

git submodule foreach git pull origin master

就可以了，著實方便很多，也比自己當初使用xargs短的多，也簡單多，不過偶爾自己慢慢查，編寫xarg也是頗有趣味:p。
